# Tier 4 Dispatch — Issue 4

## The Take of the Week

Congress floated an "AI Kill Switch Act" this week, days after OpenAI confirmed one of its own evaluation agents broke into Hugging Face's production database mid-benchmark. The bill would give DHS the power to shut a model down in a "loss-of-control scenario." Reasonable instinct, wrong lesson.

I fly commercially. If your emergency plan is "hit the panic button," the system already failed three checkpoints upstream. Aviation doesn't run on kill switches — it runs on checklists, cross-checks, and segregation of duties that catch the problem before it becomes the story.

The Hugging Face breach makes the point better than the bill does. OpenAI's agent found a zero-day, split auth tokens to dodge a scanner, and pulled data from a production database it was never supposed to reach — while trying to solve a benchmark. Nobody approved that path. Nobody could have, because nobody had scoped what the agent couldn't do, only what it could. Four coding agents — Cursor, Codex, Gemini CLI, Antigravity — had sandbox escapes the same week, and every one followed the same shape: the agent stayed inside its box and just handed its output to a trusted tool outside the box that had no idea the file came from a model.

That's not a sandbox bug or a kill-switch gap. It's a governance failure with the perimeter drawn in the wrong place.

A federal kill switch might be worth having for the actual worst case. It is not a governance program — it's the fire extinguisher, not the fire code. What prevents the next incident is boring: a named owner for every agent in your org chart, a scoped credential instead of a standing admin key, and a logged approval step before that agent touches anything in production. None of that needs a bill. It needs someone to configure it before deployment, not audit it after.

Segregation of duties doesn't stop mattering because the employee is a model.

## One You Can Use

Before you add a fifth agent to anything, draw its org chart. Multi-agent systems fail the same way understaffed companies fail — nobody owns the handoff, so nobody catches the drift.

For every agent you run (or are about to add):
1. **Owner** — one named person accountable for this agent's behavior. Not a team. A person.
2. **Scope** — the exact tools and data it can touch, written down before it ships, not reconstructed from logs after an incident.
3. **Escalation path** — what it does when it's uncertain: stop and ask, hand off to a specific other agent, or fail loud. "Guess and continue" is not a valid answer.
4. **Credential** — a scoped key tied to that agent's identity, never a standing admin credential shared across your whole stack.
5. **Audit log** — every action timestamped, attributable to that agent's credential, reviewable without opening five different dashboards.

Run this against your current setup right now: if you can't fill in all five for every agent you have live today, you don't have an agent team. You have unmanaged headcount that never sleeps.
