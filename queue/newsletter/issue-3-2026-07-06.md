# Flight Plan — Issue 3

## The Take of the Week

GPT-5.6 Sol launched this week, and everyone ran the same headline: it beats Fable 5 on Agents' Last Exam. That's not the story. The story is Terra and Luna — the smaller tiers in the same release — beating Fable 5 at roughly 1/16th the cost.

The model tier that matters for deployment isn't the flagship. It's the one that delivers 90% of the capability at a price that changes your product math. If your pricing model assumes frontier-tier costs, you're already behind the teams that ran the numbers on Terra.

The architecture decisions inside Sol matter more than the benchmark delta. "Ultra" mode coordinates four agents in parallel by default — orchestration that used to be a custom build is now a checkbox. Programmatic tool calling lets the model filter intermediate data without round-tripping every tool response through the API. Those two choices will save more real money than the next two points on a leaderboard.

Then OpenAI shipped ChatGPT Work the same week — an agent that lives inside the chat, browses your apps, stays on a project for hours, and ships finished output. Scheduled tasks are in there too. A cron job with an LLM behind it. I've been saying for a year that the cron job is the most underrated tool in AI. OpenAI just built it into the product and gave it a UI.

Here's the pattern underneath both launches: the barrier to running an agent just dropped to zero for the end user. You don't wire a harness anymore — you type what you need and an agent goes and does it, inside apps you don't control, on a schedule you didn't approve.

For any team still deciding whether to build governed agent infrastructure: your users are about to do it for you, inside ChatGPT, without your audit trail. The model got smarter. The infrastructure question — who approved this, and can you prove it — didn't get easier. It got more urgent.

## One You Can Use

Stop trusting public benchmarks to pick your model. Build a private eval instead — twenty real tasks pulled from your own production pipeline beat ten thousand synthetic ones you didn't write, and you can build the first version in an afternoon.

Steps:
1. Pull the last 20 real tasks your team actually did with (or for) an AI tool this month — real tickets, real client emails, real code diffs. Not hypotheticals.
2. For each one, write down the actual acceptable output — the thing a senior person on your team would have shipped. This is your answer key.
3. Run all 20 through each model you're considering, same prompt, same context. Score pass/fail against the answer key, not against "sounds good."
4. Re-run the same 20 tasks every time a model updates or you're evaluating a new one. Your eval only stays useful if it's a fixed yardstick against a moving target.
5. When someone pitches you a model based on a public leaderboard number, ask them to run it against your 20 tasks first. If they won't, that's your answer.

This isn't novel research — it's the same discipline aviation uses for flight test cards over manufacturer brochure numbers. The move is doing it, not admiring it.
