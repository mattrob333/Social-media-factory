The Take of the Week

The log is the agent

Four things happened this week that point to the same conclusion: we've been optimizing the wrong layer. It's not the model. It's the runtime.

First, Harrison Chase said the industry has shifted from frameworks to harnesses. Frameworks gave you vocabulary — chains, tools, prompts. Harnesses give you runtime: identity, permissions, audit, rollback. He's right, and the distinction isn't semantic. A framework is something you write to. A harness is something that runs you.

Second, Yohei Nakajima published The Log Is the Agent — a runtime architecture where the event log is the system of record and the agent is a reactive projection of that log. Your model is ephemeral; your log is permanent. If you can replay any run from its event stream, fork state at any point, and trace every artifact back to the exact model call that produced it, you have determinism, auditability, and debuggability in one decision.

Third, Rasmic shipped boringcomputers.com — spin up ephemeral compute, hand it to an agent, get an app back, and it self-destructs. Ephemeral compute + agent + self-destruct is the right pattern for most agent use cases. You don't want state you can't explain. You don't want an agent that persists its mistakes. You want a sandbox that lives exactly as long as the task and dies clean.

Fourth, the Bank of England said existing frameworks can't handle autonomous agents. The UN opened its first Global Dialogue on AI Governance in Geneva. The question from both: who approved this, and can you prove it?

The thread through all four is the same. We spent two years optimizing model capability — benchmarks, parameter counts, context windows. The models are good enough now. The bottleneck is the harness: how you feed them context, how you track what they did, how you audit their permissions, how you kill an agent that goes sideways.

Stop designing agents around the model call. Design them around the event stream. The model is the actor. The log is the agent.

    via @Rasmic
  • OpenWiki hit 7K stars in a week
    LangChain's CLI that writes and maintains a structured wiki of your codebase. The real payoff isn't productivity — it's governance. Once every agent decision traces back to a documented wiki, you have an audit trail for what the agent knew when it made each call.

  • Dario Amodei: software goes to ~zero marginal cost
    True, but incomplete. Free code has no warranty. An agent that books a wire transfer or emails a client is worthless the moment nobody can say who approved it. The model got cheap. Accountability is still expensive.

The Open-Source Swap

Trending — the infrastructure signal this week

Six repos hit GitHub trending this week that tell you exactly where agent infrastructure is headed. None of them are chatbots. The model isn't the bottleneck. The harness is where the work is happening.

  • openai/codex-plugin-cc
    One agent calling another — Codex wired into Claude Code. That's not a productivity hack, it's the org chart. Once agents delegate to agents, someone has to own the permission boundary between them.
  • alibaba/page-agent
    Browser-native GUI agent that clicks and fills forms in-page. The real win isn't "AI browses the web" — it's that you stop paying a human to do repetitive form entry across forty client accounts.
  • usestrix/strix
    Open-source AI penetration testing, past eleven hundred stars this week. Security tooling is the first place agentic AI has to prove it's trustworthy — which is the audit-trail problem all over again.
  • facebook/astryx
    Agent-ready design system — UI components built assuming an agent, not a human, will assemble the page. The frontend layer is starting to treat agents as first-class users. Real signal.
  • ogulcancelik/herdr
    Terminal-based agent multiplexer. Because when you have more than one agent, you need someone to keep them from stepping on each other.
  • JuliusBrussee/caveman
    Token-efficiency joke repo that's oddly instructive. Half of "prompt engineering" advice right now is just "stop wasting tokens."
One You Can Use

Give your agent a governance checklist

Most agent deployments skip governance entirely — no permissions model, no audit trail, no escalation path. Here's a six-question briefing you can paste into your agent tool's system slot before the next deployment. It won't fix everything, but it will catch the gap most teams don't see until an incident makes it visible.

Agent Pre-flight Briefing

"Before you take any action outside this chat, answer these six questions:

1. What data do you need, and do you have explicit permission to access it?
2. Who owns your output and can be held accountable if it's wrong?
3. Can every decision you make be traced back to a specific instruction, tool call, or model output?
4. What happens if a dependency (API, model provider, file system) fails mid-task?
5. How does a human stop you mid-operation if something looks wrong?
6. What log evidence would exist six months from now if this went wrong?

If you can't answer all six, flag the gap — don't proceed."

This works because it reframes the problem. Most agents get asked "what can you do?" and answer with capability. This briefing asks "what governs what you do?" and surfaces the gaps before they surface themselves.

Building in Public

What's shipping

This issue was assembled from the same agent pipeline that runs everything in this newsletter — bookmarks mined across 17 YouTube channels and 33 X accounts, GitHub trending scanned, each item graded by a separate grader agent against a 7-criteria rubric before it ever reaches a human for review. The full chain, from ingest to publish, runs through Hermes with separation of duties between the curator agent, the grader agent, and the human.

The pipeline itself is the build update this week. Pedigree's manifests define every agent in the chain: identity, permissions, tools, and the audit trail that ties each graded item back to its source. If you're building anything with multiple agents, you'll hit the same governance questions this pipeline answers — not in theory, but every time a grader overrules a curator or a human overrules the grader.
